> 用于简历的个人项目文档 | 2026 年

## 1. 游戏类型和核心玩法 🎮

### 游戏类型
**即时战略 (RTS) + 工厂自动化模拟游戏**

### 核心玩法
- 🏭 **工业建造系统**: 传送带物流网络、矿机开采、发电机供电、缓存箱管理
- ⚡ **电力系统**: 电网节点拓扑、电力分配、蓄电池储能
- 🎖️ **RTS 战斗**: 多阵营单位对战、英雄单位、波次进攻 AI
- 🗺️ **大地图探索**: 据点节点式关卡选择、关卡进度持久化
- 📜 **任务系统**: 目标链、剧情任务、动态事件

### 技术栈
| 类别 | 技术 |
|------|------|
| 引擎 | Unity (URP 渲染管线) |
| 语言 | C# |
| 架构 | 自定义 ECS 架构 |
| 代码规模 | 约 100+ C# 脚本文件 |

### 2.2 设计模式应用

| 模式 | 应用场景 |
|------|----------|
| **单例模式** | 所有系统管理器 (`Singleton.cs`) |

## 3. 核心贡献 ⭐

### 3.1 工业物流系统
**文件**: `Assets/Scripts/InStage/System/TransportSystem.cs`

- 🔄 实现传送带网络自动合并算法，同向传送带自动组合为 TransportLine
- 📦 使用定长数组 + 位移操作实现高效物品队列 (O(n)，n 很小)
- 🔀 支持直线对接和侧向合并两种对齐模式
- 💾 实现网络重建时完整备份/恢复传送带上物品

### 3.2 电力系统
**文件**: `Assets/Scripts/InStage/System/PowerSystem.cs`

- 🔌 使用并查集算法构建电网拓扑
- 🔍 BFS 连通性检查，避免不必要的电网重构
- ⚖️ 按满足率 (Satisfaction) 比例分配电力
- 🔋 蓄电池自动充放电补偿机制
- ⚡ 定频更新 (10Hz) 优化性能

### 3.3 寻路系统
**文件**: `Assets/Scripts/InStage/System/PathfindingSystem.cs`

- 🗺️ NavMesh 矩形剖分：贪婪算法将可行走区域合并为矩形节点
- 🔗 门户拓扑：NavPortal 记录房间连接关系
- 🧵 1D 拉绳子算法 (String Pulling) 生成最优平滑路径
- 🚪 门户时间片预约系统，解决多单位拥堵问题

```csharp
// 门户时间片预约 - 64 位掩码记录 64 个 Tick 的占用情况
public ulong[] Timetables;
public bool TryReserve(int laneIdx, int startTick, int duration, out ulong mask)
```

### 3.4 战斗系统
**文件**: `Assets/Scripts/InStage/System/AttackSystem.cs`, `AutoAISystem.cs`

- 🎯 AI 系统扫描 + 攻击系统同步的目标锁定机制
- 💥 实体子弹系统 (追踪/非追踪)
- 🩸 围棋气数系统 (GoRuleSystem) 实现地面单位呼吸规则
- 🏃 低血量单位自动溃逃机制

### 3.5 存档系统
**文件**: `Assets/Scripts/OutStage/SaveManager.cs`

- 📦 WholeComponent 深拷贝，防止运行时污染存档数据
- 🔄 热数据/冷数据分离：关卡进行中存 RAM，退出时写磁盘
- 🗂️ 使用 Newtonsoft.Json 支持 Dictionary 复杂结构序列化

### 3.6 大地图系统
**文件**: `Assets/Scripts/OutStage/BigMap/BigMapManager.cs`

- 📊 GPU 实例化渲染 (`BigMapGPUBufferManager.cs`)
- 🌐 ComputeShader 驱动的动态扩散场背景
- 🔗 节点拓扑编辑支持

### 3.7 自定义编辑器工具
**目录**: `Assets/Editor/`

- BigMapEditorWindow: 大地图节点拓扑编辑器
- MapPreviewWindow: 地图数据预览窗口
- MissionGraphWindow: 任务关系图编辑器
- LevelBakerWindow: 关卡数据烘焙工具

### 4.1 代码规模

| ECS 子系统 | 20+ |

| 自定义编辑器工具 | 4 个 |

### 4.2 性能优化成果

| 优化项 | 实现方式 | 效果 |
|--------|----------|------|
| Tick 化时间系统 | 0.1s 固定节拍 | 避免浮点误差，逻辑同步 |
| 数组位移代替 List | 传送带物品定长数组 | 减少 GC 分配 |
| Swap-back 实体删除 | O(1) 删除算法 | 高效实体管理 |
| NavMesh 局部重建 | 仅重构建筑影响区域 | 减少 80%+ 重建开销 |
| 电网智能重构 | BFS 连通性检查 | 避免不必要计算 |
| GPU 实例化渲染 | 统一 DrawCall | 支持大量单位同屏 |

### 4.3 算法复杂度

| 算法 | 复杂度 | 应用场景 |
|------|--------|----------|
| 传送带物品插入 | O(n), n≤20 | 物品传输 |
| 实体删除 | O(1) | ECS 实体管理 |
| 电网构建 | O(n·α(n)) | 并查集 |
| A* 寻路 | O(b^d) | 路径规划 |
| 门户预约 | O(1) | 位运算 |

> **MineRTS - 即时战略 + 工厂模拟游戏** | Unity3D | 独立开发者
> 
> **项目描述**: 一款融合工业自动化与 RTS 战斗的策略游戏，支持大地图探索、关卡持久化、多阵营对战。

> - 使用**策略模式**实现工业建筑工作逻辑，易于扩展新建筑类型
