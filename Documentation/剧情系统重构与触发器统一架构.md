## 📋 概述

本次重构完成了剧情系统（Story System）的核心架构搭建，并统一了 Mission 系统和 Story 系统的触发器（Trigger）实现，消除了重复代码，建立了可扩展的节点编辑器框架。

---

## 🎯 核心目标

1. **创建独立的剧情系统** - 与任务系统并列，专门处理剧情演出
2. **统一触发器架构** - Mission 和 Story 系统共用同一套 TriggerNode
3. **建立树状剧情结构** - 实现 Spine（主干）- Leaf（叶节点）的剧情编排
4. **支持一对多连接** - 节点可以触发多个后续动作

## 🎨 节点类型与连接规则

### 5 种核心节点

| 节点类型 | 颜色 | 职责 | 输入来源 | 输出去向 |
|---------|------|------|---------|---------|
| **StoryRoot** | 🟡 金色 | 剧情树起始锚点 | 无 | SpineID |
| **SpineID** | 🔵 蓝色 | 定义剧情逻辑骨架 | Root/Spine | Spine |
| **LeafID** | 🟢 绿色 | 处理具体对话演出 | Trigger | Command |
| **Trigger** | 🟣 紫色 | 监听游戏事件 | Trigger | Trigger/Leaf |
| **Command** | 🔴 红色 | 执行 RTS 动作 | Leaf/Command | Command |

### 连接白名单

```
✅ Root_Output → Spine_Input
✅ Spine_Output → Spine_Input
✅ Trigger_Output → Trigger_Input
✅ Trigger_Output → Leaf_Input
✅ Leaf_Output → Command_Input
✅ Command_Output → Command_Input

❌ 其他所有连接 = ILLEGAL
```

### Twin-ID 规则

同一个 `StoryProcessID` 必须同时拥有：
- ✅ 1 个 SpineIDNode（树主干）
- ✅ 1 个 LeafIDNode（叶演出）

保存时会校验，违反规则则阻止保存。

## 🐱 一对多连接支持

### 问题

原来的设计只支持一对一连接，无法实现：
- 一个触发器触发多个召唤点
- 一个 Leaf 节点连接多个 Command
- Spine 节点分支到多个 Spine

### 解决方案

将所有连线字段改为 `List<string>`：

```csharp
// SpineIDNodeData
public List<string> NextSpineNodeIDs;  // 一对多

// LeafIDNodeData
public List<string> ConnectedCommandNodeIDs;  // 一对多

// TriggerNodeData
public List<string> NextNodeIDs;  // 一对多

// CommandNodeData
public List<string> NextCommandNodeIDs;  // 一对多
```

### 序列化时保存连线

```csharp
// 保存一对多连线
var nextEdges = this.edges.ToList().Where(e => 
    e.output.node == node && e.output == node.OutputPort);
data.NextNodeIDs.Clear();
foreach (var edge in nextEdges)
{
    if (edge.input.node is TriggerNode nextTrigger)
        data.NextNodeIDs.Add(nextTrigger.Data.NodeID);
    else if (edge.input.node is LeafIDNode leafNode)
        data.NextNodeIDs.Add(leafNode.Data.NodeID);
}
```

### 反序列化时恢复连线

```csharp
private void RestoreConnections()
{
    var nodeMap = new Dictionary<string, BaseNode>();
    // 建立节点 ID 映射...
    
    // 恢复一对多连线
    foreach (var node in _triggerNodes)
    {
        foreach (var nextId in node.Data.NextNodeIDs)
        {
            if (nodeMap.TryGetValue(nextId, out var targetNode))
            {
                var edge = node.OutputPort.ConnectTo(targetNode.InputPort);
                AddElement(edge);
            }
        }
    }
}
```

## 📊 压缩优化（额外工作）

### StoryData.cs 地图数组压缩

将地图数组（`groundMap`、`gridMap`、`effectMap`）改为 Gzip + Base64 压缩存储：

```csharp
public string groundMapBase64
{
    get
    {
        // int[] → byte[] → Gzip 压缩 → Base64
        byte[] rawBytes = new byte[groundMap.Length * 4];
        Buffer.BlockCopy(groundMap, 0, rawBytes, 0, rawBytes.Length);
        
        using var output = new MemoryStream();
        using (var gzip = new GZipStream(output, CompressionLevel.Optimal))
        {
            gzip.Write(rawBytes, 0, rawBytes.Length);
        }
        return Convert.ToBase64String(output.ToArray());
    }
    set { /* 反向操作 */ }
}
```

**压缩效果**: 300KB → 3KB（100 倍压缩率，因为地图数据重复度高）

### WholeComponent 稀疏序列化

使用 `SparseArrayConverter<T>` 只序列化非 default 的实体：

```csharp
[JsonConverter(typeof(SparseArrayConverter<CoreComponent>))]
public CoreComponent[] coreComponent;
```

**JSON 格式**:
```json
{
  "count": 1024,
  "entries": [
    {"i": 0, "d": {...}},  // 只存非 default 的实体

  ]
}
```

## 📝 技术要点总结

### 1. 统一架构

- 共用 `TriggerNode` 类（Mission 和 Story 系统）
- 统一 `TriggerNodeData` 数据类
- 统一命名规范（触发器节点）

### 2. 一对多连接

- 所有连线字段改为 `List<string>`
- 序列化时遍历所有边保存
- 反序列化时遍历 ID 列表恢复

### 3. Twin-ID 规则

- Spine 和 Leaf 必须成对出现
- 保存时校验，违反则阻止

### 4. 端口兼容性

- 移除 `portType` 检查
- 只检查方向和节点

### 5. 压缩优化

- 地图数组：Gzip + Base64
- 组件数组：稀疏序列化
