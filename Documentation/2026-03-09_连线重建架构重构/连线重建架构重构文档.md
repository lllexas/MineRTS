## 过去的痛点

### 1. 连线数据分散管理

每个节点类型有自己的连线字段，没有统一格式：

```csharp
// MissionData.cs
public string NextMissionID;      // 下一个任务 ID
public string RewardID;           // 奖励 ID

// MapNodeData.cs
public List<string> ConnectedSpawnIds;    // 连接的召唤节点
public List<string> ConnectedBrainIds;    // 连接的 AI 节点

// TriggerNodeData.cs
public List<string> NextNodeIDs;  // 下一个节点列表
```

**问题**: 字段命名不统一，有的是单个 ID，有的是列表，无法用通用逻辑处理。

---

### 2. 连线恢复逻辑重复

每个 GraphView 子类都要手动写连线恢复代码：

```csharp
// MissionGraphView.cs
protected override void RestoreConnections(Dictionary<string, BaseNode> nodeMap)
{
    foreach (var missionNode in _missionNodes)
    {
        if (!string.IsNullOrEmpty(missionNode.TypedData.RewardID) &&
            nodeMap.TryGetValue(missionNode.TypedData.RewardID, out var rewardTarget))
        {
            var edge = missionNode.OutputPort.ConnectTo(((RewardNode)rewardTarget).InputPort);
            AddElement(edge);
        }
        // ... 更多重复逻辑
    }
}

// StoryGraphView.cs
protected override void RestoreConnections(Dictionary<string, BaseNode> nodeMap)
{
    // 类似的逻辑又要写一遍...
}
```

**问题**: 代码重复，新增节点类型需要修改多处，容易遗漏。

---

### 3. 输入端口没有明确定义

节点只声明了输出端口（通过字段值），但没有声明输入端口：

```csharp
public class MissionData : BaseNodeData
{
    public string NextMissionID;  // 输出：指向下一个任务
    // 但是输入呢？谁连接到我？没有定义！
}
```

**问题**: 读取连线数据时，无法知道目标节点的哪个输入端口应该被设置。

---

### 4. 基类设计不合理

`BaseGraphView` 使用 `Nodes` 列表来创建 nodeMap，但子类实际使用专用列表：

```csharp
// BaseGraphView.cs
protected void RestoreConnections()
{
    var nodeMap = new Dictionary<string, TNode>();
    foreach (var node in Nodes)  // 子类的节点不在这个列表里！
    {
        // ...
    }
}

// MissionGraphView.cs
private List<MissionNode> _missionNodes;   // 实际节点在这里
private List<MapNode> _mapNodes;
// ...
```

**问题**: 基类的模板方法无法工作，子类必须重写整个方法。

## 方案的原理

### 核心思想：端口标签 + 统一连线数据

每个节点数据类明确声明：
1. **我的 ID** - `NodeID`（继承自 `BaseNodeData`）
2. **我的输入端口** - 使用 `[InPort(index)]` 标签标记字段
3. **我的业务参数** - 普通字段，无标签
4. **我的输出端口** - 使用 `[OutPort(index)]` 标签标记字段

```csharp
[Serializable]
public class MissionData : BaseNodeData
{
    // 1. 我的 ID（继承自 BaseNodeData）
    // public string NodeID;
    
    // 2. 我的输入端口
    [InPort(0)]
    public string TriggerID;           // 触发此任务的触发器 ID
    
    [InPort(1)]
    public string PreviousMissionID;   // 前置任务 ID
    
    // 3. 我的业务参数
    public string Title;
    public MissionPriority Priority;
    public List<MissionGoal> Goals;
    
    // 4. 我的输出端口
    [OutPort(0)]
    public string NextMissionID;       // 下一个任务 ID
    
    [OutPort(1)]
    public string RewardID;            // 奖励节点 ID
    
    // 5. 统一的连线数据（由系统自动生成）
    public List<ConnectionData> OutputConnections;
}
```

---

### 连线数据结构

```csharp
[Serializable]
public struct ConnectionData
{
    public int FromPortIndex;      // 输出端口索引
    public string TargetNodeID;    // 目标节点 ID
    public int ToPortIndex;        // 输入端口索引
}
```

**JSON 示例**:
```json
{
  "NodeID": "mission_1",
  "Title": "主线任务 1",
  "NextMissionID": "mission_2",
  "RewardID": "reward_1",
  "OutputConnections": [
    { "FromPortIndex": 0, "TargetNodeID": "mission_2", "ToPortIndex": 1 },
    { "FromPortIndex": 1, "TargetNodeID": "reward_1", "ToPortIndex": 0 }
  ]
}
```

---

### 数据流

**保存时**:
```
业务字段值 → 反射读取 [OutPort] 字段 → 生成 ConnectionData → 写入 JSON
(NextMissionID="mission_2")         (FromPortIndex=0, TargetNodeID="mission_2")
```

**读取时**:
```
JSON → ConnectionData → 查找目标节点 → 设置 [InPort] 字段 → 恢复编辑器连线
                         (ToPortIndex=1)   (PreviousMissionID="mission_1")
```

```csharp
    }
```

### 4. 收集连线数据方法

**文件**: `Assets/Scripts/Editor/NekoGraph/Core/BaseGraphView.cs`

```csharp
/// <summary>
/// 收集连线数据 - 遍历带 [OutPort] 标签的字段生成连线数据喵~
/// 支持 string 字段和 List<string> 字段喵~
/// </summary>
protected static List<ConnectionData> CollectConnections(BaseNodeData nodeData)
{
    var connections = new List<ConnectionData>();
    var type = nodeData.GetType();
    
    // 遍历所有带 [OutPort] 标签的字段
    foreach (var field in type.GetFields(BindingFlags.Public | BindingFlags.NonPublic | BindingFlags.Instance))
    {
        var outPortAttr = field.GetCustomAttribute<OutPortAttribute>();
        if (outPortAttr == null) continue;

        var fieldValue = field.GetValue(nodeData);
        
        // 处理 string 字段喵~
        if (fieldValue is string stringId && !string.IsNullOrEmpty(stringId))
        {
            connections.Add(new ConnectionData(
                outPortAttr.Index,
                stringId,
                0  // 默认连接到目标节点的输入端口 0
            ));
        }
        // 处理 List<string> 字段喵~
        else if (fieldValue is List<string> stringList && stringList.Count > 0)
        {
            foreach (var id in stringList)
            {
                if (!string.IsNullOrEmpty(id))
                {
                    connections.Add(new ConnectionData(
                        outPortAttr.Index,
                        id,
                        0
                    ));
                }
            }
        }
    }

    return connections;
}
```

### 5. 恢复连线方法

**文件**: `Assets/Scripts/Editor/NekoGraph/Core/BaseGraphView.cs`

```csharp
/// <summary>
/// 恢复连线 - 根据 ConnectionData 恢复所有节点的连线喵~
/// 同时设置目标节点的 [InPort] 字段值
/// </summary>
protected static void RestoreConnections<TGraph>(
    TGraph graph,
    Dictionary<string, BaseNode> nodeMap) where TGraph : BaseGraphView
{
    foreach (var kvp in nodeMap)
    {
        var node = kvp.Value;
        var data = node.Data;

        if (data.OutputConnections == null || data.OutputConnections.Count == 0) continue;

        foreach (var conn in data.OutputConnections)
        {
            if (string.IsNullOrEmpty(conn.TargetNodeID)) continue;
            if (!nodeMap.TryGetValue(conn.TargetNodeID, out var targetNode)) continue;

            // 获取输出端口
            var outputPort = GetPortByIndex(node, conn.FromPortIndex, Direction.Output);
            if (outputPort == null) continue;

            // 获取输入端口
            var inputPort = GetPortByIndex(targetNode, conn.ToPortIndex, Direction.Input);
            if (inputPort == null) continue;

            // 连接
            var edge = outputPort.ConnectTo(inputPort);
            graph.AddElement(edge);
            
            // 设置目标节点的 [InPort(ToPortIndex)] 字段值为源节点的 ID 喵~
            SetInPortFieldValue(targetNode.Data, conn.ToPortIndex, node.Data.NodeID);
        }
    }
}

/// <summary>
/// 设置目标节点的 [InPort] 字段值喵~
/// </summary>
private static void SetInPortFieldValue(BaseNodeData data, int portIndex, string sourceNodeID)
{
    var type = data.GetType();
    foreach (var field in type.GetFields(BindingFlags.Public | BindingFlags.NonPublic | BindingFlags.Instance))
    {
        var inPortAttr = field.GetCustomAttribute<InPortAttribute>();
        if (inPortAttr == null || inPortAttr.Index != portIndex) continue;

        // 处理 string 字段喵~
        if (field.FieldType == typeof(string))
        {
            field.SetValue(data, sourceNodeID);
        }
        // 处理 List<string> 字段喵~
        else if (field.FieldType == typeof(List<string>))
        {
            var list = field.GetValue(data) as List<string>;
            if (list == null)
            {
                list = new List<string>();
                field.SetValue(data, list);
            }
            if (!list.Contains(sourceNodeID))
            {
                list.Add(sourceNodeID);
            }
        }
        break;
    }
}
```

```csharp
public override MissionPackData SerializeToPack()
{
    var pack = new MissionPackData();
    
    // 更新所有节点数据并生成连线数据
    foreach (var node in _missionNodes) 
    { 
        node.UpdateData(); 
        node.TypedData.OutputConnections = CollectConnections(node.TypedData); 
    }
    // ... 其他节点类型同理
    
    pack.Nodes = new List<BaseNodeData>();
    foreach (var node in _missionNodes) 
        pack.Nodes.Add(node.TypedData);
    // ...
    
    return pack;
}
```

**反序列化时**:
```csharp
public override void PopulateFromPack(MissionPackData pack)
{
    // 清空并创建节点...

    // ... 其他节点类型同理
    
    // 调用基类的通用方法恢复连线
    RestoreConnections(this, nodeMap);
}
```

### 给节点数据类添加端口标签

```csharp
[Serializable]
public class MapNodeData : BaseNodeData
{
    public string MapID;
    public Vector2Int SelectedPosition;
    
    // 输入端口
    [InPort(0)]
    public string FromMissionID;       // 来自哪个任务

    // 输出端口（列表类型）
    [OutPort(0)]
    public List<string> ConnectedSpawnIds = new List<string>();
    
    [OutPort(1)]
    public List<string> ConnectedBrainIds = new List<string>();
}
```

### 自动生成的连线数据

假设一个地图节点连接了 2 个召唤节点和 1 个 AI 节点：

```csharp
// 原始字段值
ConnectedSpawnIds = { "spawn_1", "spawn_2" }
ConnectedBrainIds = { "ai_1" }

// CollectConnections 生成的连线数据
OutputConnections = 
{
    ConnectionData { FromPortIndex=0, TargetNodeID="spawn_1", ToPortIndex=0 },
    ConnectionData { FromPortIndex=0, TargetNodeID="spawn_2", ToPortIndex=0 },
    ConnectionData { FromPortIndex=1, TargetNodeID="ai_1", ToPortIndex=0 }
}
```

---

### 恢复后的效果

读取 JSON 后，调用 `RestoreConnections`:

1. 找到 `spawn_1` 节点
2. 设置 `spawn_1.FromMapID = "map_1"`（根据 `[InPort(0)]` 字段）
3. 在编辑器中连接 `map_1` 的输出端口 0 → `spawn_1` 的输入端口 0
