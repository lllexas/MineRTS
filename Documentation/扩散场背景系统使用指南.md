# 扩散场背景系统 - 使用指南

## 概述

扩散场背景系统是一个基于**密度梯度扩散模型**的炫酷背景效果。使用 ComputeShader 模拟密度的注入、扩散和衰减过程，输出高质量的 RenderTexture 供后续处理。

## 核心原理

### 密度梯度扩散模型

```
∂ρ/∂t = D·∇²ρ - R·ρ + S

其中：
- ρ    : 密度场
- D    : 扩散系数（Diffusion coefficient）
- ∇²   : 拉普拉斯算子（扩散项）
- R    : 衰减率（Decay rate）
- S    : 源项（节点注入）
```

### 视觉流程

```
节点注入 → 密度扩散 → 衰减 → 融合 → 输出 RenderTexture

每帧循环：
Frame 1:  ● (注入)
Frame 2:  ● → ~~~ (扩散)
Frame 3:  ● → ~~~~~ (继续扩散 + 新注入)
Frame 4:  ●   ● → ~~~~~~~ (多节点融合)
```

## 快速开始

### 步骤 1：在 Unity 中打开项目

打开 Unity Editor，让 Unity 编译新创建的文件。

### 步骤 2：配置 ViewportBackgroundQuad

1. 在 Hierarchy 中找到 `ViewportBackgroundQuad` 物体
2. 确保它作为相机的子物体存在

### 步骤 3：创建 DiffusionFieldBridge

1. 在 `ViewportBackgroundQuad` 下创建一个空子物体，命名为 `DiffusionField`
2. 将 `DiffusionFieldBridge.cs` 脚本挂载到该物体上

### 步骤 4：配置 DiffusionFieldBridge

在 Inspector 中配置以下参数：

```
ComputeShader 设置:
  - ComputeShader: 拖入 DiffusionField.compute

扩散参数:
  - Diffusion Coeff: 0.1 (扩散系数)
  - Decay Rate: 0.02 (衰减率)
  - Inject Strength: 1.0 (注入强度)
  - Inject Radius: 0.05 (注入半径)

纹理设置:
  - Texture Width: 512
  - Texture Height: 512

模式激活配置:
  - Active In Modes: 添加 Default (或其他需要的模式)
```

### 步骤 5：配置材质

1. 选中 `DiffusionDisplay.mat` 材质
2. 将材质拖到 `ViewportBackgroundQuad` 的 Mesh Renderer 组件的 Material 槽中

### 步骤 6：运行测试

1. 确保场景中有 `BigMapGPUBufferManager` 存在
2. 确保大地图数据已加载
3. 点击 Play，查看扩散效果

## 参数调节指南

### 扩散参数

| 参数 | 默认值 | 说明 | 调节建议 |
|------|--------|------|----------|
| **Diffusion Coeff** | 0.1 | 扩散系数，越大扩散越快 | 0.05-0.2 之间效果最佳 |
| **Decay Rate** | 0.02 | 衰减率，越大密度消失越快 | 0.01-0.05 之间 |
| **Inject Strength** | 1.0 | 注入强度，越大节点处密度越高 | 0.5-3.0 之间 |
| **Inject Radius** | 0.05 | 注入半径（世界空间） | 根据节点大小调整 |

### 效果参考

```
扩散系数对比：
小 (0.05): 密度扩散慢，边界清晰
中 (0.1):  扩散速度适中，推荐
大 (0.3):  扩散快，密度场均匀

衰减率对比：
小 (0.01): 密度残留久，拖尾长
中 (0.02): 适中，推荐
大 (0.05): 密度快速消失，需要持续注入

注入强度对比：
小 (0.5): 节点处密度低
中 (1.0): 适中，推荐
大 (3.0): 节点处密度很高，扩散范围大
```

## 技术细节

### ComputeShader 工作流程

```
每帧：
1. 从 BigMapGPUBufferManager 获取节点数据
2. 设置 ComputeShader 参数（扩散系数、衰减率等）
3. 设置 Ping-Pong 输入/输出缓冲
4. Dispatch CSDiffuseInject Kernel
   - 读取 InputBuffer (上一帧密度)
   - 计算 3x3 拉普拉斯卷积（扩散）
   - 应用衰减
   - 在节点位置注入新密度
   - 写入 OutputBuffer
5. 交换 Ping-Pong 缓冲
6. 将 OutputBuffer 设置到材质
```

### 拉普拉斯卷积核

```
卷积核结构（3x3）：
| 0.05  0.2  0.05 |
| 0.2  -1.0   0.2  |
| 0.05  0.2  0.05 |

物理意义：
- 四个方向邻居权重 0.2
- 四个对角邻居权重 0.05
- 中心权重 -1.0
- 总和 = 0（守恒）
```

### Ping-Pong 缓冲原理

```
Frame t:
  读取：Buffer_A (上一帧密度)
  计算：扩散 + 衰减 + 注入
  写入：Buffer_B (新密度)

Frame t+1:
  读取：Buffer_B
  计算：扩散 + 衰减 + 注入
  写入：Buffer_A

优势：
- 时间连续性
- 密度场自然演化
- 无需每帧重新计算
```

## 性能优化

### ComputeShader 性能

- **Dispatch 时间**: ~0.3-0.8ms (512x512)
- **内存占用**: ~1MB (2 张 ARGBHalf 纹理)
- **FPS 影响**: < 1 帧

### 优化建议

1. **降低分辨率**: 512 → 256 (性能提升 4 倍)
2. **使用合并 Kernel**: `CSDiffuseInject` 比分离版本快
3. **调整线程组**: 确保 threadGroupSize 与 ComputeShader 一致

## 常见问题

### Q1: 看不到扩散效果？

**检查清单：**
1. BigMapGPUBufferManager 是否存在并已初始化
2. 大地图数据是否已加载（有节点数据）
3. ComputeShader 是否正确分配
4. 扩散系数是否太小（建议 0.1）
5. 注入强度是否足够（建议 1.0）

### Q2: 扩散太快/太慢？

**调节建议：**
- 太快：降低 Diffusion Coeff (0.1 → 0.05)
- 太慢：提高 Diffusion Coeff (0.1 → 0.2)

### Q3: 密度场闪烁或异常？

**可能原因：**
1. Ping-Pong 缓冲未正确初始化 → 点击 ForceReinitialize
2. 纹理格式不支持 → 确认使用 ARGBHalf
3. ComputeShader 编译错误 → 查看 Console 窗口

### Q4: 性能过低？

**优化建议：**
1. 降低纹理分辨率（512 → 256）
2. 降低扩散系数（减少计算量）
3. 使用更小的线程组（8 → 4）

## 扩展开发

### 添加新的扩散模式

在 `DiffusionField.compute` 中添加：

```hlsl
#pragma kernel CSDiffuseAdvanced

[numthreads(8,8,1)]
void CSDiffuseAdvanced(uint3 id : SV_DispatchThreadID)
{
    // 自定义扩散逻辑
    // 例如：各向异性扩散、多物质扩散等
}
```

在 `DiffusionFieldBridge.cs` 中选择使用哪个 Kernel。

### 添加颜色映射

在显示 Shader 中添加：

```hlsl
// 根据密度值映射不同颜色
float3 colorA = float3(0, 0.5, 1);  // 低密度 - 蓝色
float3 colorB = float3(1, 0.5, 0);  // 高密度 - 橙色
float3 finalColor = lerp(colorA, colorB, density);
```

### 添加噪声扰动

在扩散计算中添加：

```hlsl
float noise = tex2D(_NoiseTex, uv).r;
newDensity += noise * _NoiseStrength;
```
