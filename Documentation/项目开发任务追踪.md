### 任务 #8：编写大地图底层Shader渲染
**状态**：已完成 (2026-03-01)
**优先级**：中
**创建时间**：2026-03-01
**目标**：为大地图编写底层Shader，提供视觉增强效果。

**实现内容**：
1. 创建`ViewportBackgroundQuad.cs`：动态调整3D Quad大小以完全覆盖正交相机视口，实现无限世界空间网格底图
2. 创建`InfiniteWorldGrid.shader`：基于世界空间坐标的网格绘制Shader，支持ComputeBuffer节点/边数据渲染
3. 创建`BigMapGPUBufferManager.cs`：将BigMapData转换为GPU缓冲区，供Shader读取节点位置、连线等数据
4. 集成到BigMapManager：大地图打开时自动更新GPU缓冲区数据

**需求**：
1. 基于"UI Toolkit交互层 + Shader底图层"混合架构，实现底层地图渲染

3. 需要与现有的UI Toolkit渲染层协同工作
4. 可能包括地形高度图、网格线、区域着色等效果

**技术考虑**：
1. 使用Unity Shader Graph或编写自定义Shader
2. 考虑与UI Toolkit的z-order和渲染顺序
3. 性能优化，避免与UI Toolkit渲染冲突
4. 支持动态更新（如区域变化、战争迷雾等）

**关键文件**：
- `Assets/Scripts/OutStage/BigMap/ViewportBackgroundQuad.cs`（新建）
- `Assets/Shaders/InfiniteWorldGrid.shader`（新建）
- `Assets/Scripts/OutStage/BigMap/BigMapGPUBufferManager.cs`（新建）
- `Assets/Scripts/OutStage/BigMap/BigMapManager.cs`（修改，添加GPU缓冲区更新）

**验收标准**：
1. Shader能够正确渲染底层地图元素
2. 与UI Toolkit节点渲染层无缝结合

### 任务 #10：完善导航网格(NavMesh)系统
**状态**：待开始
**优先级**：中高 (影响核心玩法)
**创建时间**：2026-03-01
**目标**：修复拆除建筑后NavMesh未刷新的bug，确保寻路系统正确响应地形变化。

**背景**：
根据bug报告《2026-02-27_拆除建筑后navmesh未刷新_寻路单位未重新寻路.md》，当前存在严重问题：
1. 建筑拆除后，其占据的网格区域没有被正确标记为可通行
2. NavMesh未相应刷新，路网拓扑未更新
3. 正在寻路的单位不会重新计算路径，导致卡住或绕行失败

**问题分析**：
1. **建筑类型判断问题**：`(unitType & UnitType.Building) != 0`可能未正确识别所有建筑类型
2. **RebuildNavMesh实现缺陷**：可能未正确处理"移除障碍"的情况，矩形剖分算法可能在某些边缘情况下失败
3. **寻路系统未响应变化**：PathfindingSystem可能没有订阅NavMesh更新事件，单位路径缓存了旧的通行性信息
4. **网格占据清理不彻底**：GridSystem.Instance.ClearOccupantRect可能没有正确清理所有相关网格

**相关文件**：
- `Assets/Scripts/InStage/System/EntitySystem.cs:271-351`（建筑拆除逻辑，第298-306行调用RebuildNavMesh）
- `Assets/Scripts/InStage/System/GridSystem.cs:706-752`（RebuildNavMesh方法实现）
- `Assets/Scripts/InStage/System/PathfindingSystem.cs`（需要检查寻路响应机制）
- `Assets/Scripts/InStage/Component/CoreComponent.cs`（UnitType定义）

**修复方向**：
1. **确保NavMesh正确刷新**：
   - 验证RebuildNavMesh在建筑拆除时总是被调用
   - 确保矩形剖分算法正确处理"移除障碍"场景
   - 验证网格占据清理的完整性

2. **寻路系统响应机制**：
   - 实现NavMesh变化事件系统
   - 让PathfindingSystem监听变化并重新计算受影响单位的路径
   - 添加路径定期验证机制

3. **类型系统增强**：
   - 统一建筑类型判断逻辑
   - 添加IsBuilding()辅助方法确保一致性

**调试建议**：
1. 启用Gizmos可视化NavMesh，观察建筑拆除前后网格变化
2. 在DestroyEntity中添加调试日志，确认RebuildNavMesh被调用
3. 创建测试场景验证建筑拆除后区域IsWalkableStatic返回true
4. 测试单位能否成功寻路通过原建筑区域

**验收标准**：
1. 建筑拆除后，该区域立即变为可通行，NavMesh正确刷新
2. 正在寻路的单位自动重新计算路径，顺利通过原建筑区域
3. 无需玩家手动重新下达移动指令
4. 系统性能不受显著影响
