# 任务系统运行简介

## 概述

MineRTS的任务系统是一个**事件驱动的剧本架构**，允许设计者创建复杂的任务链、剧情事件和战斗波次。系统核心特性包括：

- **可视化任务编辑**：通过图形化编辑器设计任务关系网
- **多类型目标**：建造、击败、生存时间、赚钱等多种目标类型
- **导演系统**：基于时间或任务完成的事件触发器
- **奖励机制**：金钱、科技点、蓝图解锁等奖励发放
- **连环任务**：任务完成后自动激活后续任务

## 系统组件

### 1. 核心运行时组件

| 组件 | 位置 | 职责 |
|------|------|------|
| **MissionManager** | `Assets/Scripts/OutStage/Mission/MissionManager.cs` | 任务生命周期管理、进度追踪、奖励发放 |
| **DirectorSystem** | `Assets/Scripts/InStage/System/DirectorSystem.cs` | 剧本事件触发与执行（时间/任务完成触发） |
| **PostSystem** | 游戏事件总线 | 各系统间的事件通信 |

### 2. 数据模型

```csharp
MissionPackData                     // 任务包（一个关卡或剧本）
├── Missions: List<MissionData>     // 任务列表
│   ├── Goals: List<MissionGoal>    // 任务目标（建造、击败等）
│   ├── Reward: MissionReward       // 奖励（金钱、科技点、蓝图）
│   └── NextMissionID: string       // 连环任务链接
├── Rewards: List<RewardSaveData>   // 独立奖励节点
├── ScenarioEvents: List<ScenarioEventData> // 导演事件节点
├── SpawnActions: List<SpawnActionData>     // 单位召唤节点
└── AIBrainActions: List<AIBrainActionData> // AI控制节点
```

### 3. 可视化编辑器

- **位置**: `Assets/Editor/MissionGraphWindow.cs`
- **打开方式**: Unity Editor菜单 `Tools/猫娘助手/剧本编辑器 (Graph Mode)`
- **功能**: 拖拽式任务图编辑、节点连接、JSON序列化

## 运行流程

### 1. 任务加载

```csharp
// 在游戏启动时调用
MissionManager.Instance.LoadMissionPack("Missions/Tutorial");

// 内部流程：
// 1. 从 Resources/Missions/Tutorial.json 加载JSON
// 2. 反序列化为 MissionPackData
// 3. 缝合奖励数据（RewardID → Reward）
// 4. 添加到 ActiveMissions 列表
// 5. 通知UI刷新
// 6. 导演系统加载剧本事件
```

### 2. 进度追踪

任务进度通过PostSystem事件驱动更新：

| 游戏事件 | 对应目标类型 | 监听器 | 处理逻辑 |
|----------|--------------|--------|----------|
| `"建筑完成"` | BuildEntity | MissionManager | 建造计数+1 |
| `"击败目标"` | KillEntity | MissionManager | 击败计数+1 |
| `"出售资源"` | SellResource | MissionManager | 出售资源计数+1 |
| `"金币更变"` | EarnMoney | MissionManager | 赚钱金额累计 |
| `"生存时间增加"` | SurviveTime | MissionManager | 生存时间累计 |

### 3. 任务完成

```csharp
MissionManager.CompleteMission(mission)
├── 标记任务为已完成
├── 发放奖励（通过EVT_MISSION_REWARD事件）
├── 激活连环任务（NextMissionID）
├── 发送UI_MISSION_COMPLETE事件
└── 检查关卡胜利条件（所有主线任务完成）
```

### 4. 导演事件触发

导演系统支持两种触发方式：

**时间触发**：
```csharp
// 在Update()中监测时间
if (TimeSystem.Instance.TotalElapsedSeconds >= targetTime) {
    TriggerEvent(event);
}
```

**任务完成触发**：
```csharp
// 监听UI_MISSION_COMPLETE事件
[Subscribe("UI_MISSION_COMPLETE")]
private void OnMissionCompleted(object data) {
    // 查找等待该任务完成的事件
    TriggerEvent(relatedEvent);
}
```

**事件执行流程**：
1. 导演事件触发 → 查找关联的召唤节点
2. 批量召唤单位 → 收集生成的EntityHandle
3. 如有AI节点 → 挂载AI控制到召唤单位

## 可视化编辑器使用指南

### 1. 基本操作

| 操作 | 说明 |
|------|------|
| **右键菜单** | 在画布空白处右键 → 选择节点类型创建 |
| **空格键** | 快速呼出节点创建面板 |
| **拖拽连接** | 从节点的输出端口拖到其他节点的输入端口 |
| **框选复制** | Ctrl+C复制选中节点 → Ctrl+V粘贴（位置偏移） |
| **保存/加载** | 工具栏按钮保存为JSON / 从JSON加载 |

### 2. 节点类型

| 节点类型 | 颜色 | 功能 |
|----------|------|------|
| **任务节点** | 深蓝 | 定义任务目标、奖励、优先级 |
| **奖励节点** | 深绿 | 独立奖励定义（金钱、科技点、蓝图） |
| **导演节点** | 紫色 | 事件触发器（时间/任务完成触发） |
| **召唤节点** | 深红 | 单位召唤配置（兵种、数量、位置） |
| **AI挂载节点** | 青蓝 | AI波次控制（目标坐标、标识符） |

### 3. 连接规则

```
任务节点:
  - 输出端口 → 下一任务节点 (逻辑链)
  - 奖励端口 → 奖励节点 (奖励连接)

导演节点:
  - 输入端口 ← 任务节点 (触发条件)
  - 输出端口 → 召唤节点 (触发行为)

召唤节点:
  - 输出端口 → AI挂载节点 (AI控制)
```

## 集成与扩展

### 1. UI系统集成

- **任务追踪UI**: `UIMissionTracker` 监听 `UI_MISSION_REFRESH` 事件
- **任务项UI**: `UIMissionItem` 显示单个任务进度

- 奖励发放触发全局进度更新
- 蓝图解锁保存到玩家档案

public enum GoalType { BuildEntity, KillEntity, ..., NewGoalType }

[Subscribe("新游戏事件")]

    UpdateGoalProgress(GoalType.NewGoalType, key, amount);

// 3. 在UIMissionItem中添加显示名称

    case GoalType.NewGoalType: return "新目标";

## 性能注意事项

1. **事件频率**: 高频事件（如"生存时间增加"）应适当合并批量处理
2. **对象池**: MissionArgs使用64容量的对象池减少GC
3. **UI刷新**: 只在进度实际变化时发送UI刷新事件
4. **导演系统**: 时间触发在Update中只检查未触发事件

## 常见问题

### 1. 任务未激活
- 检查连环任务的 `NextMissionID` 是否正确链接
- 确认任务 `IsActive` 属性为true

### 2. 进度不更新
- 确认PostSystem事件名称与监听器匹配
- 检查目标类型 `GoalType` 和 `TargetKey` 是否正确
- 查看控制台是否有类型错误日志

### 3. 导演事件不触发
- 确认触发类型（Time/MissionCompleted）设置正确
- 检查触发参数（时间数值/任务ID）是否正确
- 查看事件 `HasTriggered` 状态

### 4. 单位未召唤
- 检查召唤节点的 `BlueprintId` 是否存在
- 确认 `SpawnPos` 在地图可放置区域
- 查看AI节点连接是否正确

## 总结

MineRTS的任务系统提供了一个灵活、可视化的剧本设计框架：

- **策划友好**: 图形化编辑器降低配置复杂度
- **扩展性强**: 易于添加新目标类型和触发器
- **性能优化**: 事件驱动架构减少不必要的更新
- **集成完整**: 与UI、存档、AI系统深度集成

通过合理使用任务链、导演事件和奖励机制，可以设计出从简单教程到复杂战役的多样化游戏内容。
